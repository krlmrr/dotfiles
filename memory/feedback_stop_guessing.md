---
name: Stop guessing, listen to the user
description: Don't cycle through random fixes - listen to what the user is telling you and think before acting
type: feedback
---

Stop guessing and cycling through random fixes. When the user says something isn't working, actually think about WHY before trying the next random thing.

**Why:** Went in circles for 20+ attempts trying to fix a dock gap issue, repeatedly trying the same broken approaches (sleep timers, different delays, AppleScript vs defaults write) instead of listening to the user and understanding the root cause. The user had to get increasingly frustrated before the actual fix was found.

**How to apply:** When a fix doesn't work, don't just tweak the same approach. Step back, understand what's actually happening, and try a fundamentally different approach. If the user says "it's not a timing issue" — believe them.
