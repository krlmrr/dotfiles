## Tailscale = the access layer (tailnet-only, no public exposure)
- Official Tailscale **catalog app**, running **host-networking** mode → the TrueNAS host itself is on the tailnet as node **`wozserver`** = `wozserver.tail01309a.ts.net` = `100.80.17.60`. Host services (UI:443, SMB:445) answer on that name directly.
- ALSO advertises subnet **`192.168.1.0/24`** (approved) → every LAN device reachable over the tailnet via its LAN IP. Redundant-but-good with host-net.
- Tailnet domain `tail01309a.ts.net` (same as Synology era). **Disable key expiry** on the node so the box never drops off (verify done).
- Access: UI `https://wozserver` · `ssh karlm@wozserver` · `smb://wozserver`.
- mDNS is **`truenas-discoveryd`** (not avahi) — advertises `.local` on LAN interfaces only. `.local` does NOT traverse the tailnet, so for off-LAN use you must address the **MagicDNS FQDN** explicitly.
