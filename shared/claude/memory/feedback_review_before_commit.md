Always review all changes against the general, laravel, and vue coding skills BEFORE committing or pushing. Never commit first and review after.

**Why:** A pre-commit hook requires skill review before continuing. Reviewing after the commit violates the required ordering.

**How to apply:** After making changes and before running `git commit`, explicitly review each changed file against the applicable skills. Only then proceed with the commit.
